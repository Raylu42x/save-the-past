Use these commands to demonstrate the ingest, upload, and monitoring flow.

1. The "Magic" Trigger (Manual Test)

Run this to create a dummy drive folder and trigger the upload service.
mkdir -p /data/ingest/DEMO_DRIVE_001
echo '{"ingest": "complete", "upload": "pending"}' > /data/ingest/DEMO_DRIVE_001/status.json

2. Watch the Brain (Service Logs)

Show the live logs as the script detects the new folder and starts Rclone.
sudo journalctl -u upload_service.service -f

3. Status Dashboard

The URL to show on your phone or tablet during the demo.
http://autocopy.local:5000

4. Emergency Fixes

If you get a "Permission Denied" error during the demo:
sudo chown -R pi:pi /data/ingest && sudo chmod -R 755 /data/ingest
To restart the engine:
sudo systemctl restart upload_service.service

5. Verify Cloud

Check if the files actually landed in Google Drive via CLI:
rclone lsd gdrive:DataSuperHero_Uploads
